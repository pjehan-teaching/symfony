Mailer & Messenger

Envoyer des e-mails

Le composant Mailer permet d'envoyer des e-mails via un transport configurable :

composer require symfony/mailer

Le transport est défini par la variable d'environnement MAILER_DSN dans le fichier .env :

# SMTP classique
MAILER_DSN=smtp://user:pass@smtp.example.com:587

# Fournisseur tiers (ex : Brevo, Mailgun...) via un pont dédié
MAILER_DSN=brevo+api://KEY@default
Mailer

Composer et envoyer un e-mail

On injecte MailerInterface puis on construit un objet Email :

use Symfony\Component\Mailer\MailerInterface;
use Symfony\Component\Mime\Email;

public function contact(MailerInterface $mailer): Response
{
    $email = (new Email())
        ->from('no-reply@example.com')
        ->to('user@example.com')
        ->subject('Bienvenue !')
        ->text('Version texte du message')
        ->html('<p>Version <strong>HTML</strong> du message</p>');

    $mailer->send($email);

    return $this->redirectToRoute('home');
}

💡 En développement, le Profiler intercepte les e-mails (onglet « E-mails ») : ils ne sont pas réellement envoyés.

E-mails avec Twig

La classe TemplatedEmail permet d'utiliser un template Twig comme corps du message :

use Symfony\Bridge\Twig\Mime\TemplatedEmail;

$email = (new TemplatedEmail())
    ->from('no-reply@example.com')
    ->to($user->getEmail())
    ->subject('Confirmez votre inscription')
    ->htmlTemplate('emails/registration.html.twig')
    ->context([
        'user' => $user,
        'confirmationUrl' => $url,
    ]);

$mailer->send($email);
composer require symfony/twig-bundle  # requis pour TemplatedEmail

Le problème du traitement synchrone

Envoyer un e-mail peut prendre plusieurs secondes (connexion au serveur SMTP). Si cela se produit pendant une requête HTTP, l'utilisateur attend devant sa page.

La solution : déléguer ces tâches lentes à un traitement asynchrone. On répond immédiatement à l'utilisateur, et le travail est effectué en arrière-plan.

C'est le rôle du composant Messenger.

Messenger

Messenger : message et handler

composer require symfony/messenger

Un message est un simple objet transportant des données :

namespace App\Message;

class SendWelcomeEmail
{
    public function __construct(
        public readonly int $userId
    ) {
    }
}

Un handler (attribut #[AsMessageHandler]) traite le message :

namespace App\MessageHandler;

use App\Message\SendWelcomeEmail;
use Symfony\Component\Messenger\Attribute\AsMessageHandler;

#[AsMessageHandler]
class SendWelcomeEmailHandler
{
    public function __construct(private MailerInterface $mailer) {}

    public function __invoke(SendWelcomeEmail $message): void
    {
        // ... récupérer l'utilisateur et envoyer l'e-mail
    }
}

Dispatcher un message

On injecte MessageBusInterface et on envoie le message sur le bus :

use Symfony\Component\Messenger\MessageBusInterface;

public function register(MessageBusInterface $bus, User $user): Response
{
    // ... enregistrer l'utilisateur

    // Réponse immédiate ; l'e-mail partira en arrière-plan
    $bus->dispatch(new SendWelcomeEmail($user->getId()));

    return $this->redirectToRoute('home');
}

Sans transport configuré, le handler s'exécute immédiatement (synchrone). C'est la configuration du transport qui rend le traitement asynchrone.

Transport asynchrone

Un transport définit où les messages sont mis en file d'attente (base de données, Redis, RabbitMQ...) :

# .env — file d'attente stockée dans la base de données
MESSENGER_TRANSPORT_DSN=doctrine://default
# config/packages/messenger.yaml
framework:
    messenger:
        transports:
            async: '%env(MESSENGER_TRANSPORT_DSN)%'
        routing:
            App\Message\SendWelcomeEmail: async

Un worker consomme ensuite la file en continu (typiquement lancé par Supervisor en production) :

php bin/console messenger:consume async -vv

Récapitulatif du flux asynchrone

Contrôleur
   │  $bus->dispatch(new SendWelcomeEmail($id))
   ▼
Bus Messenger ──► Transport "async" (file en base de données)
                          │
                          ▼
                  messenger:consume (worker en arrière-plan)
                          │
                          ▼
                  Handler ──► Mailer ──► e-mail envoyé

L'utilisateur reçoit sa réponse sans attendre l'envoi de l'e-mail.
